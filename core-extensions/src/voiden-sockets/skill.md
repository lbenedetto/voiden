## Extension: Voiden Sockets / gRPC

Provides WebSocket and gRPC block types via `socket-request`, `surl`, `smethod`, `proto`, `grpc-messages-node`.

### socket-request — gRPC/WebSocket Request Container

```yaml
---
type: socket-request
attrs:
  uid: "uid"
content:
  - type: smethod
    attrs:
      uid: "uid"
      method: UserService.GetUser
      visible: true
    content: grpc.UserService.GetUser
  - type: surl
    attrs:
      uid: "uid"
    content: "grpcs://{{GRPC_HOST}}:443"
  - type: proto
    attrs:
      uid: "uid"
      service: UserService
      method: GetUser
      file: user.proto
---
```

### smethod — gRPC Method

```yaml
- type: smethod
  attrs:
    uid: "uid"
    method: UserService.GetUser    # ServiceName.MethodName
    visible: true
  content: grpc.UserService.GetUser
```

### surl — gRPC / WebSocket URL

```yaml
- type: surl
  attrs:
    uid: "uid"
  content: "grpcs://api.example.com:443"    # grpc:// or grpcs:// or ws:// or wss://
```

### proto — Protobuf Definition

```yaml
- type: proto
  attrs:
    uid: "uid"
    service: UserService
    method: GetUser
    file: user.proto
```

### grpc-messages-node — gRPC Messages

Handles all gRPC call types with messages:

```yaml
---
type: grpc-messages-node
attrs:
  grpcId: "grpc-123"
  callType: unary               # unary | server-streaming | client-streaming | bidirectional
  service: UserService
  method: GetUser
  target: "grpcs://api.example.com:443"
content:
  - type: message
    attrs:
      messageType: request
      value: '{"id": "{{USER_ID}}"}'
---
```

**Call types:**
| `callType` | Description |
|------------|-------------|
| `unary` | Single request → single response |
| `server-streaming` | Single request → stream of responses |
| `client-streaming` | Stream of requests → single response |
| `bidirectional` | Stream of requests → stream of responses |
